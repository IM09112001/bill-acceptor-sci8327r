Задание: купюроприемник SCI8327R
RS232 + WebSocke1. Что должно работать
подключение к купюроприемнику через RS232;
WebSocket сервер для приема команд;
прием JSON-команд от клиента;
отправка команд в купюроприемник;
ответ клиенту в JSON;
отправка событий всем подключенным клиентам, когда что-то происходит с купюрой или устройством;
нормальная обработка ошибок, без падения программы.
2. WebSocket
WebSocket должен быть доступен по адресу:
ws://localhost:8080
Если адрес, порт или serial port будут другими, это должно настраиваться через config или .env.
3. Формат входящего запроса
Команды должны приходить в WebSocket в таком формате:
{"device":"BILL_ACCEPTOR","method":"OPEN","id":"open-cli"}
device всегда BILL_ACCEPTOR;
method - название команды;
id - id запроса, его надо вернуть обратно в ответе;
params используется только там, где нужны дополнительные параметры.
4. Команды
Команда
Что должна делать
OPEN
Открыть подключение к купюроприемнику. Нужно открыть RS232 порт и проверить, что устройство отвечает.
CLOSE
Закрыть подключение к купюроприемнику. Нужно остановить работу с serial port и освободить порт.
STATUS
Вернуть текущее состояние устройства. Должно быть понятно: подключено устройство или нет, какой сейчас режим, есть ли купюра в ожидании.
CAPTURE
Включить прием купюр. После этой команды устройство должно ждать купюру.
STACK
Принять купюру и отправить ее в кассету. Работает только если купюра сейчас находится в escrow.
RETURN
Вернуть купюру обратно пользователю. Работает только если купюра сейчас находится в escrow.
AUTOSTACK
Включить или выключить автоматический прием. enabled true - купюра сразу уходит в кассету. enabled false - купюра ждет STACK или RETURN.
ENABLE
Разрешить прием купюр.
DISABLE
Запретить прием купюр.
5. События от устройства
События - это не ответ на команду. Это сообщения, которые сервер сам отправляет всем клиентам, когда состояние устройства поменялось.
CONNECTED - устройство подключилось;
DISCONNECTED - устройство отключилось;
ESCROW - купюра вставлена и ждет решения;
STACKING - купюра уходит в кассету;
STACKED - купюра принята;
RETURNING - купюра возвращается;
RETURNED - купюра возвращена;
REJECTED - купюра отклонена;
ERROR - ошибка устройства или программы.
6. Ошибки, которые надо обработать
port  устройство не найден;
купюроприемник не отвечает;
пришел неправильный JSON;
device не BILL_ACCEPTOR;
неизвестный method;
STACK или RETURN отправили, но купюры в escrow нет;
застряла купюра;
кассета полная;
кассета снята;
устройство вернуло ошибку.
7. RS232
использовать параметры, которые нужны для SCI8327R / SCL8327R;
port должен задаваться через настройки;
для Linux порт может быть /dev/ttyUSB0, /dev/ttyACM0;
для Windows порт может быть COM1, COM2, COM3 и так далее;
если устройство отключилось, программа не должна просто падать.
